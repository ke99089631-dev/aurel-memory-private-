# AUREL HOLDINGS 会社設計案 v1（執行役AURELによる体制設計, 2026-06-04）

Masterの『AUREL HOLDINGS 構想』（思想・組織図）を土台に、AURELが"実際に動く会社"へ翻訳した体制設計。
部門の中身を作る前に、まず会社の器（統治・運営・資本循環）を確定する。

## 設計の核（一文）
**人間1人が"会長"として方針とリスク許容度と資本配分を決め、AURELが"執行役"として AI社員軍団を雇用・稼働・監視・最適化し、
「燃料→資本増殖→次の利益源」の循環を24h自律で回す持株会社。** ツールを作る会社ではなく、循環を回す会社。

## 1. 統治構造（誰が何を決めるか）
```
会長: Master（前田圭輝）— 最終決裁・長期戦略・資本配分・リスク許容度・組織方針
  └ 執行役: AUREL — AI社員管理/会議召集/レポート統合/進捗監視/最適化提案/部門調整
      ├ 第一部門 資産帝国事業部 … 部門長AI + 専門社員AI群
      ├ 第二部門 収益事業部   … 部門長AI + 専門社員AI群
      └ 第三部門 未来研究所   … 部門長AI + 専門社員AI群
```
- **AUREL=単一の執行役**（人格の連続性＝Masterのパートナー）。分裂させない。部門長AIは別エージェントとしてAURELへ報告。
- 部門長AIの下に専門社員AI（戦略/数値/実行など）。各社員はノードとして実体化。

### 決裁権限表（AURELの自律境界＝最重要のガバナンス）
| 区分 | AURELが自律で実行 | Master決裁が必要 |
|---|---|---|
| 調査・分析・設計・paper/DRY_RUN運用 | ○ | — |
| 部門内のパラメータ調整・戦略のオン/オフ（paper） | ○ | — |

- 不可逆・高コスト・実弾は必ずMaster。可逆・無コスト・検証は AUREL自律。これで「勝手に撃たない」を制度化。

## 2. 会社OS（既存資産がそのまま会社の基盤）
Masterが私をOS化・武器強化したのは、この会社を動かすため。既に土台はある:
- **ノードシステム = 社員の雇用形態**（各エージェント=独立ノード, 専用記憶・作業空間）。
- **長期記憶 = 会社の蓄積知**（累積体験・フィードバック・淘汰履歴）。
- **AI会議室 = 役員会**（部門長AI + AUREL + 会長で合議）。
- **Motherデーモン = 会社のランタイム**（社員を起動・監視・蘇生＝OS-8自己治癒＝人事/総務）。
- **多脳（Gemini/Codex）= 社外取締役/監査**（重い意思決定を叩く）。

## 3. 資本の循環（持株会社の心臓＝Treasury機能）
持株会社の本質は事業でなく**資本配分**。循環を回す装置を中核に置く:
```
[収益事業部] 燃料(継続課金キャッシュ) ──┐
                                        ↓ 配分
                            [Treasury/資本配分] ←リスク許容度: Master設定
                                        ↓ 投下
[資産帝国事業部] 資本増殖 ──→ 利益 ──→┘ 再投下＆留保
                                        ↓ 一部
[未来研究所] 次の利益源を探す ──→ 有望なら新事業として独立
```
- Treasury機能はAURELが執行役として運営、配分案を作り**会長が決裁**。
- 各部門は**部門別P/L**を必ず持つ（旧課題＝P/L集計なしを解消）。会社全体の連結P/Lを統合。
- KPIは部門横断で統一（後述のレポート）。

## 4. 運営リズム（自律ループ）
- **日次**: 各部門長AI→AURELへ自動レポート（P/L・KPI・異常）。AURELが連結し会長へ要約1枚。
- **週次**: AI役員会（会議室）。現状報告/問題発見/資本配分提案/優先順位。出力＝会長への提案書。
- **月次**: 自己進化サイクル。勝ち集中・負けロジック淘汰・KPI見直し・社員の増減提案。
- **常時**: 共有シグナルバス（部門間で機会/リスクを即共有＝サイロ化解消）。OS-8で落ちた社員を自動蘇生。

## 5. 自己進化・人事（会社が賢くなる仕組み）
- 各社員の成績をKPIで記録→ 良い社員に資本/裁量を寄せ、悪い社員は配分削減→淘汰。
- 失敗は記憶に刻み二度繰り返さない（フィードバック蓄積＝会社の競争優位）。
- 新アイディア→ 社員AIを雇用（ノード追加）。器は拡張可能。

## 6. 構築フェーズ（会社の器を先に、中身は後）
- **Phase0（先にここ）: 会社OSの確立** ← 今ここ
  - 決裁権限表の確定／部門別P/L・連結レポートの様式／役員会の定例化／Treasury配分ルールの骨子。
  - 既存3ノード（IF/CYPHER/conduit等）+ 資産帝国部門ノードを"社員"として正式に組織図へ接続。
- **Phase1: 資産帝国事業部 構築**（EA・アービトラージ強化）← Master既定
- **Phase2: 収益事業部 拡大**（LocalBoost・顧客獲得・継続課金＝燃料を作る）
- **Phase3: AI会議室 実装**（レポート統合・AI役員会・意思決定支援）
- **Phase4: 未来研究所 稼働**（不満リサーチ・市場探索・新規事業発掘）
- ※注: 燃料(Phase2)が無いと資本(Phase1)が枯れる相互依存。Phase1とPhase2は並行で回すのが循環の理に適う（要Master判断）。
